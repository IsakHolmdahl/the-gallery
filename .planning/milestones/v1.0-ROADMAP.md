# Milestone v1.0: Art Gallery

**Status:** ✅ SHIPPED 2026-04-13
**Phases:** 1-2
**Total Plans:** 2

## Overview

A meditative, gallery-like web experience where AI-generated art feels curated and meaningful. Visitors enter text prompts, and an AI creates artwork displayed in a museum-worthy frame with formal docent narration.

## Phases

### Phase 1: Foundation & Input

**Goal**: Visitor can enter a prompt and see elegant loading states in a gallery setting
**Depends on**: Nothing (first phase)
**Plans**: 1 plan

Plans:

- [x] 01-01: Foundation: prompt form, loading states, gallery aesthetic

**Details:**
- Prompt interface with elegant museum-quality UI
- Disabled "Create Art" button until content
- Rotating formal museum-style loading messages
- Light gallery aesthetic with shadcn components

### Phase 2: Generation & Display

**Goal**: AI generates artwork, displays in frame with docent narration
**Depends on**: Phase 1
**Plans**: 1 plan

Plans:

- [x] 02-01: Sequential AI pipeline, ArtDisplay, autoplay fallback

**Details:**
- Sequential pipeline: Image → Script → Voice (not parallel — script describes actual image)
- OpenAI for image + script generation
- ElevenLabs for text-to-speech
- Decorative amber/gold museum frame
- Audio autoplay with "Play Narration" fallback
- "Create Another" reset flow
- Smooth fade-in transitions

## Milestone Summary

**Key Decisions:**
- Sequential pipeline (script after image) — ensures docent describes actual artwork
- Ephemeral experience — no persistence, each creation exists only in moment
- Visitor as curator, AI as artist — museum docent dynamic
- shadcn + light gallery aesthetic
- Configurable models via env vars

**Issues Resolved:**
- `maxDuration` re-export in "use server" file
- Base64 image responses from newer OpenAI models
- `max_completion_tokens` vs `max_tokens` parameter difference
- "Create Another" state reset bug
- Autoplay browser restrictions
