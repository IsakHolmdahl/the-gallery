# Milestone v1.2: Image Upload & Lecture

**Status:** ✅ SHIPPED 2026-04-20
**Phases:** 5-7
**Total Plans:** 3
**Quick Tasks:** 3

## Overview

Let visitors drop their own image into the gallery and get a docent narration, skipping the AI image generation step. Visitors can now upload photos of real artwork and receive AI-powered museum-style narration with voice synthesis.

## Phases

### Phase 5: Vision Pipeline

**Goal**: Server can generate docent narration from an uploaded image, with voice synthesis
**Depends on**: Phase 4
**Plans**: 1 plan

Plans:

- [x] 05-01: Vision server action (createArtFromImage with validation)

**Details:**
- Added `createArtFromImage` server action to `src/app/actions.ts`
- Calls OpenAI vision API with `detail: "high"` for richer art analysis
- Generates docent narration voice via ElevenLabs TTS pipeline
- Input validation: format allowlist (PNG/JPEG/WEBP), size cap, data URI structure check
- Reuses existing `DOCENT_SYSTEM_PROMPT` (not duplicated)
- Security: validation before any API calls (no unnecessary costs)

### Phase 6: Upload Interface

**Goal**: Users can upload images via drag-and-drop or click, see a preview, and submit for narration
**Depends on**: Phase 5
**Plans**: 1 plan

Plans:

- [x] 06-01: Upload interface (drop zone, preview, dynamic button, mode switching)

**Details:**
- Created `file-utils.ts` with `validateFile` function for client-side image validation
- Drop zone overlay on textarea with amber/stone border glow on drag-over
- Plus icon indicator when dragging files
- Image preview in simplified gallery frame
- X button to clear preview and return to text mode
- Click-to-upload via hidden file input
- Dynamic button text: "Create Art" → "Analyze art" when image present
- Dual-mode submission: `createArt` for text, `createArtFromImage` for images
- Object URL cleanup on unmount (no memory leaks)

## Milestone Summary

**Key Decisions:**
- Reuse OPENAI_SCRIPT_MODEL for vision (no new env var)
- GIFs rejected entirely (OpenAI doesn't support animated GIFs)
- `detail: "high"` for richer docent narration
- Drop zone overlays textarea with amber/stone glow
- Simplified gallery frame for preview
- Button text changes to "Analyze art" when image present
- X button only for clearing preview
- Counter-based drag detection (no flicker)
- Canvas API compression (zero dependencies) for Vercel compatibility
- 1-second voice delay for better reveal timing

**Issues Resolved:**
- Drag-enter/leave flicker when moving over child elements
- Non-image drag payloads triggering false drop zones
- Vercel free tier 1.5MB body limit (increased to 10mb + client compression)
- Volume range error with floating point precision in background music hook
- Voice starting immediately on image reveal (added 1s delay)

**Technical Debt:**
- None — all requirements satisfied
