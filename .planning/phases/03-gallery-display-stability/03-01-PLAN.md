**Root cause (D-07):** `useActionState` serializes previous response state (including base64 audio) back to server on each submission, exceeding Vercel's 1MB body limit on second generation.

**In `src/app/actions.ts`:**
1. Change `createArt` signature from `useActionState` pattern to simple async function:
   - Remove `prevState` parameter
   - Change to: `export async function createArt(formData: FormData): Promise<{...}>`
   - Keep the same return type and body logic
2. Change image size from `"1024x1024"` to `"1792x1024"` (D-01, D-02)

**In `src/components/prompt-form.tsx`:**
1. Remove `useActionState` import, add `useTransition` import
2. Replace `const [state, formAction, pending] = useActionState(createArt, null)` with:
   - `const [state, setState] = useState<{ success: boolean; error?: string; imageUrl?: string; audioBase64?: string; script?: string } | null>(null)`
   - `const [pending, startTransition] = useTransition()`
3. Create a form submit handler that calls `startTransition` wrapping `createArt(formData)` then `setState(result)`
4. In `onReset` callback: call `setState(null)` in addition to `setShowResult(false)` and `setFormKey(prev => prev + 1)` — ensures no previous submission data carries forward (D-09)
5. Attach `onSubmit` handler to form instead of `action={formAction}`, use `e.preventDefault()` + `new FormData(e.currentTarget)`
  </action>
