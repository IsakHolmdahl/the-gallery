The `createArt` action pattern:
1. Validate input
2. Generate image → get imageUrl
3. Generate script via openai.chat.completions.create → get script text
4. Generate voice via elevenlabs.textToSpeech.stream → get audioBase64
5. Return { success, imageUrl, audioBase64, script }

The new `createArtFromImage` follows the same pattern but:
- Step 1: Receives base64 image directly (no generation)
- Step 2: Calls openai.chat.completions.create with vision (image_url content type)
- Step 3: Same ElevenLabs TTS call
- Step 4: Returns same shape, using original base64 data URI as imageUrl
