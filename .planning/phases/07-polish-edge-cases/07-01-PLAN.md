## Key Context from Phase 6

The upload interface is in `src/components/prompt-form.tsx`. Current issues:

1. **Drag flicker:** Uses simple `isDragging` boolean. `handleDragLeave` sets it to `false`, but moving between child elements (the plus icon SVG, the span) fires leave→enter→leave→enter. Fix: counter-based approach.

2. **No non-image filtering:** `handleDragOver` accepts everything. Only `handleDrop` filters via `processFile`. Fix: check `dataTransfer.items` types in `handleDragOver`.

3. **Object URLs:** Cleanup on unmount exists (line 106-112), and `clearImage` revokes (line 97-98). But `clearImage` is also called from the `onReset` callback in ArtDisplay (line 169). Verify this path works — it does (clearImage is called before formKey increment).

4. **Error display:** Simple `<p className="text-sm text-red-600">` with no retry action. Server errors from `createArt`/`createArtFromImage` are caught and returned as `{success: false, error: string}`.

5. **Glow animation:** Phase 6 used static Tailwind classes (`bg-amber-50/80 border-amber-400 ring-amber-200/50`). DROP-05 requires smooth animated glow. Tailwind v4 dropped `animate-pulse`. Need custom CSS `@keyframes` in globals.css.

```tsx
const handleDragOver = (e: React.DragEvent) => {
  e.preventDefault()
  e.stopPropagation()
  // Check if drag payload contains files/images — ignore text/link drags
  const hasImage = Array.from(e.dataTransfer.items).some(
    item => item.kind === 'file' && item.type.startsWith('image/')
  )
  if (hasImage || e.dataTransfer.types.includes('Files')) {
    setDragCounter(prev => prev + 1)
  }
}
```

const handleDragLeave = (e: React.DragEvent) => {

}
