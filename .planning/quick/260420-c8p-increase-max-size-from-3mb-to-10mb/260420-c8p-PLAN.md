**Important note:** The original 3MB limit existed because Vercel serverless has a ~4.5MB request body limit, and base64 inflates ~33%. A 10MB raw image becomes ~13.3MB base64, which exceeds this. This change assumes the deployment target supports larger payloads (e.g., Vercel Pro with increased body limit, or non-Vercel deployment). If deploying on Vercel free tier, an API route migration with `bodyParser: { sizeLimit: '10mb' }` may be needed as a follow-up.
</context>
